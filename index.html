Semantic Segmentation

Pixel-wise image classification.

Pixel-wise image classification is a well-studied problem in computer vision. The task of semantic image segmentation is to classify every pixel in the image.

The key idea in this project is to implement a “fully deep convolutional neural network” that takes input of arbitrary size, understands the input at pixel-level and produce a good segmentation map as an output. It has been implemented using a prominent state-of-the-art architecture called U-Net.

DATASET: The dataset used is called Cityscapes dataset. It focuses on semantic understanding of urban street scenes. It consists of 30 different classes and out of them only 6 classes namely car, road, person, sky, building, tree and the rest of the classes are considered as background.

Languages and Libraries: Python, TensorFlow, OpenCV, Numpy, PIL, Matplotlib and Tensorboard.

Spam Classifier

Classifies an Email into Spam or Ham.

The goal of this project is to use Natural Language Processing(NLP) and machine learning and build an algorithm to classify an Email as 'Spam' or 'Ham' without using any external libraries.

A machine learning model called Naive Bayes Classifier is used in this project. It is a probabilistic model used for classification task and is based on bayes theorem.

DATASET: The Spam/Ham email data with labels provided by an open-source platform called Spam Assassin is used in this project. It consists of 5,799 emails, labeled as spam or ham accordingly.

Languages and Libraries: Python, Numpy, Pandas, Matplotlib, NLTK and WordCloud.

Visual Search and Similarity

Recommends similar catalog images based on input image.

The goal of this project is to build a deep learning system that takes an input image and outputs similar images based on given input image. It has been implemented using convolutional neural networks, deep metric learning and few-shot learning.

The aim here is to learn a mapping from images to feature space where distances directly correspond to their similarities and dissimilarities. This is achieved using triplet loss.

DATASET: Street2shop dataset has been used in the project to match real-world garment item to the same item in the catalog. It contains street2shop pairs for 11 different clothing categories - bags - belts - dresses - eyewear - footwear - hats - leggings - outerwear - pants - skirts - tops. Here we focus on only one category i.e, 'dresses'.

Languages and Libraries: Python, TensorFlow, OpenCV, PIL, Numpy and Tensorboard.

Sentiment Analysis

Sentiment analysis of twitter tweets.

Sentiment analysis uses Natural Language Processing (NLP) to make sense of text data and machine learning to automatically identify and classify subjective information from .

The goal of this project is to implement a machine learning algorithm to detect the sentiment in twitter tweets. It takes a tweet as an input and outputs the sentiment of the tweet i.e, classifies the tweet as 'positive' or 'negative'. It uses TF-IDF for feature extraction of the text data for the tweets and different standard classifiers like Random forest, Logistic regression and SVM. The performance of various classifiers are compared using standard classification metrics.

DATASET: The dataset used is a public dataset called Sentiment140. It consists of 1.6 million twitter tweets extracted using twitter api. The tweets are annotated (0 = negative, 4 = positive) and they can be used to detect the sentiment.

Languages and Libraries: Python, NLTK, Scikit-learn, Numpy, and Pandas.

NVIDIA Triton Inference Server

A machine learning inference platform.

This project is about building an inference server for deloying and running machine learning models using NVIDIA Triton Inference Server. It is an open source inference serving platform that lets you deploy trained AI models from any framework (TensorFlow, TensorRT, PyTorch, ONNX Runtime, or a custom framework), from local storage or Google Cloud Platform or AWS S3 on any GPU- or CPU-based infrastructure (cloud, data center, or edge).

The server provides an infernce service via HTTP or gRPC endpoints, allowing remote clients to request inferencing for any model being managed by the server.

Essen Markt

A web application for food sharing.

Roughly one-third of the food produced in the world for human consumption every year approximately 1.3 billion tonnes gets lost or wasted. It wouldn’t be exaggerating if food waste is called a catastrophic problem globally today. When observed meticulously, it can be seen that most of the discarded food is still edible. When the technological advancements today are made in every field, why not as well have an application where tonnes of food can be saved from going to the bin where more than half of this could be eaten.

As a solution to the food wastage problem, this project is a simple web-based application where locals can share surplus food with another. Raw, cooked, uncooked, unopened; as long as the food is edible the food can be shared. All one must do is let others know that there is leftover food and put it on the application and someone will pick it up. One can decide to give away for free or for a little price. It is a simple application, where you have to add the leftover food and someone will pick it up and the surplus food is saved and not thrown away.

View this project on GitHub: https://github.com/Rathna21/Essen-Markt-PM102-WS1718
